03 ‚Äî The Playbook
10 Tips to Become a Top 1% User
01

Tests First, Always

Gary ran 3 days without tests, then hit 100% coverage in 1 day and "sped up like crazy." Agent output quality jumps dramatically when it can verify its own work. Tests = evals.

02

Clear Context at 50%

Don't let the agent spiral. When token usage crosses ~50%, reset. The model in a poisoned context is like a student with 5 minutes left on an exam ‚Äî it panics.

03

Keep Codebases Small

Target 100‚Äì200 lines per focused package. Use microservices. Less plumbing = better agent performance. Let Vercel/Next.js handle boilerplate.

04

Let Agents Check Their Work

Lint, CI, test suites ‚Äî the more automated verification loops you give agents, the better their output. Code review bots (Reptil, Cursor, Codex) on every PR.

05

Think Like a Manager

Top users direct flows, kick off parallel tasks, review results, and context-switch. You're the architect; agents are your build crew.

06

Use the 10-Minute Pockets

No more 4-hour blocks needed. Kick off a task between meetings, let it run, review later. The maker-vs-manager divide is dissolving.

07

Give It Real Environment Access

CLI agents can access your dev DB, production DB, job queues. Gary debugged nested delayed jobs 5 levels deep ‚Äî agent found the bug and wrote a test.

08

Run Code Review Aggressively

Use multiple review bots: Reptil for correctness, Cursor bug bot for speed, Codex for deep review. Don't merge without automated review.

09

Prototype ‚Üí Production Pipeline

Write quick in-memory prototypes, then tell the agent to make it production-ready. Great for validating ideas fast before investing architecture time.

10

Keep Tinkering ‚Äî It Changes Monthly

Calvin switches tools every few months. The landscape moves fast. The people who benefit most are those constantly pushing the boundary of what's possible.
